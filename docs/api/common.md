## 响应格式

所有接口统一返回格式：

```json
{
  "code": 200,      // 状态码：200成功，400业务失败，500系统错误
  "message": "提示信息",
  "data": {}        // 返回数据（可选）
}
```

## 错误码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 400 | 业务失败（如参数错误、验证码错误等） |
| 401 | 未鉴权或 Token 无效 |
| 500 | 系统错误（服务器内部错误） |

## 常见错误

### 参数验证失败
```json
{
  "code": 400,
  "message": "参数验证失败：邮箱格式不正确"
}
```

### 验证码错误
```json
{
  "code": 400,
  "message": "验证码错误或已过期"
}
```

### 用户已存在
```json
{
  "code": 400,
  "message": "该邮箱已被注册"
}
```

### 登录失败
```json
{
  "code": 400,
  "message": "邮箱或密码错误"
}
```

### Token无效
```json
{
  "code": 400,
  "message": "登录已过期，请重新登录"
}
```

## 静态资源访问

上传成功的文件可通过以下路径直接访问：

| 路径前缀 | 说明 | 示例 |
|----------|------|------|
| `/static/avatars/<filename>` | 用户/群组头像 | `http://localhost:8080/static/avatars/abc.jpg` |
| `/static/files/<filename>`   | 普通上传文件 | `http://localhost:8080/static/files/report.pdf` |

## 开发建议

1. **请求超时**: 建议设置15秒超时
2. **Token存储**: 登录成功后将 token 存储在 localStorage 中
3. **错误处理**: 统一处理 400 和 500 错误，给用户友好提示
4. **验证码**: 验证码有效期3分钟，请提示用户及时使用
5. **密码安全**: 前端不应明文存储密码
6. **WebSocket 重连**: 建议实现指数退避重连策略，避免短时间内大量重连请求
7. **文件上传**: 上传前应在前端进行文件类型和大小校验，减少无效请求
